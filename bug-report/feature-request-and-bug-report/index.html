기능 요청 및 버그 신고

알고 계셨나요?

노드 운영자는 기능 요청을 작성하고 Github에서 버그 신고를 제출하여 노드 실행 경험을 개선할 수 있습니다. 문제를 열려면 Github의 nearcore 레퍼지토리로 이동하여 이슈를 작성합니다. 이때 이슈에 nodeX 태그를 붙여서 해당 이슈가 Node Experience와 연관되어 있다는 것을 나타내세요.

기능 요청 및 버그 신고​

NEAR 팀은 노드 및 밸리데이터 커뮤니티의 피드백을 적극적으로 요청하고 있으며, 커뮤니티가 기능/개선 요청에 참여하고 버그 보고서를 제출할 수 있는 프로세스를 제공합니다. 기존 NEAR 밸리데이터 채널 외에도, NEAR 팀은 기능 요청 및 버그 신고에 대한 공식 프로세스를 도입하고 있습니다.

NEAR 노드 운영 경험과 관련하여, 모든 버그 및 기능 개선 사항은 Github 프로젝트 추적기 Node Experience에서 공개적으로 추적됩니다. 노드 운영자라면, 기능 및 버그에 대한 새로운 이슈를 생성하고, 이러한 문제를 Github 프로젝트 추적기에 추가해 주세요.

  • 새로운 기능 / 개선 요청:

    • Github issue를 생성하세요.
    • 이슈는 Github 프로젝트 Node Experience 내 Incoming Requests 열을 통해 검토 및 필터링되며, 우선 순위에 따라 개발에 반영됩니다.

  • 버그 신고:

    • 노드 운영자 문제의 경우, https://near-node.zendesk.com/hc/en-us/requests/new에서 Node 팀에 대한 Zendesk 이슈를 만들고 로그를 첨부해 보세요.

질문이 있으신가요?

StackOverflow에 물어보세요!
